Silas-Creed Franchise Architecture

The Bottleneck

Managing narrative "State-Drift" across a 500k+ word IP ecosystem. Maintaining canonical consistency regarding character characterization, plot-critical pivots—such as the Thorne betrayal—and timeline-continuity is physically impossible using standard linear drafting tools.

The Architectural Solution

I built a relational state-machine to manage the franchise IP. By tracking plot-critical flags and character-arc metadata as database entries rather than simple prose, I created a "Source of Truth" that prevents canonical violations during the drafting phase. This architecture allows me to map complex character arcs against time-stamped narrative segments, ensuring that every branch of the story remains structurally sound and internally consistent.

Command Stack

Airtable (RDBMS) | Metadata Flag-Mapping | Canonical Version Control | VS Code
